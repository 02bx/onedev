Branches Can Be Rebased

Declare branches which can be rebased. This setting tells integration process that these branches have rebase capability, and they will be rebased if relevant integration strategy defines the option rebase if possible. Leave this option undefined if no branches can be rebased in this repository.

Downstream Integration Strategies

Define strategies on how to integrate pull requests sent from downstream branches. Target branch of the pull request will be used to find the first matching strategy to apply to the integration. If no matching strategy is found, the default strategy will apply. The pull request can be either rebased or merged based on strategy options:
  • If option rebase if possible is specified and source branch (ie. downstream branch) can be rebased, the source branch will be rebased to target branch, and then target branch will be fast forwarded to source branch; otherwise, the pull request will be merged following below procedure.
  • If pull request has to be merged, the option merge always determines whether or not to create a merge commit even if target branch can be fast forwarded to source branch.
