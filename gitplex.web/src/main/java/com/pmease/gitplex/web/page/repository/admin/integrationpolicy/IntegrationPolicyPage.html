Integration Policies

Integration policies defines applicable strategies when integrate pull requests into target branches. For instance besides the ordinary merge strategy, one can use strategy rebase strategy to rebase pull request commits on top of target branch, or use squash strategy to squash all commits in a pull request into one before merging with target.

For a given pull request, its branch information will be matched against branch criterias of integration policies defined here, and the first matching policy will be used. If no matching policy can be found, the pull request will use strategy merge always.
